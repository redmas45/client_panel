# Client Panel

Client Panel is the tenant-facing portal for AI Salesman Hub clients. It shows scoped analytics, usage, catalog status, conversations, and token policy for one client at a time.

The app is intentionally separate from both the Hub and the storefront:

- `AI_salesman_plugin` owns the backend data, authentication API, analytics API, and token policy API.
- `Vercel_website` owns the AI-KART storefront and shared public Nginx route.
- `client_panel` owns only the client-facing React/Vite UI.

Client Panel is a Node/Vite frontend. It does not use a Python backend or host venv.

## Core Features & Product Role

Client Panel gives store owners a limited, secure view into their AI assistant across five dedicated tabs:

1. **Overview Tab**:
   - **Today's Brief**: Automated daily briefing displaying top product demand signals, token quota runway projections, catalog readiness ratios, and peak traffic patterns.
   - **Since Last Visit**: A browser-local snapshot telemetry system (`localStorage` based) that tracks metrics delta (turns, sessions, token usage, top product) since the merchant's last dashboard visit.
   - **Shopper Question Simulator**: An interactive tool allowing store owners to test hypothetical customer queries, previewing their intent classification and recommended actions.
   - **Executive Store-Manager Summary**: AI briefing generated by AI Hub or heuristics.
   - **Token Snapshot**: Real-time token usage progress and session limit config shortcuts.
2. **Demand Tab**:
   - **Interactive Trend Charts**: Interactive bar graphs tracking search demand (turns) and token consumption.
   - **Assistant Health**: Telemetry on Action rate, Error rate, Latency distribution, and Transport mix.
   - **Rankings**: Lists detailing product demand, intent mix, and status mix.
3. **Conversations Tab**:
   - **Intent Inbox**: Classifies customer queries into 5 primary intent buckets (*Ready to buy*, *Price sensitive*, *Delivery or policy*, *Needs recommendation*, *Weak answer risk*) with specialized guidelines.
   - **Conversation Log**: Comprehensive shopper session list with full user-to-AI turn details, latency, intent, and token cost.
4. **Catalog Tab**:
   - **Catalog Health**: Snapshot of total products, active products, and category grouping counts.
   - **Catalog Opportunity Map**: Maps search demand frequency to product indexes, pointing out merchandising recommendations for high-interest inventory items.
5. **Token Policy Tab**:
   - **Session Token Limit**: Configurable limits preventing single shopper sessions from exhausting the storefront's token pool.
   - **Token Runway Forecast**: Multi-day runway predictions calculated against recent traffic paces, plus remaining sessions estimations.

It must not expose cross-tenant Hub data.

## Hub API Contract

Client Panel reads and writes scoped data through AI Hub APIs under:

```text
/v1/client-panel/*
```

Hub is expected at:

```text
http://143.198.5.97/aihub
```

AI Hub must provide:

- Client login by `site_id` and password.
- Scoped analytics.
- Scoped usage and token policy.
- Scoped catalog summary.
- Scoped conversation list/details.

AI Hub `.env` must include:

```env
CLIENT_PANEL_DEFAULT_PASSWORD=choose_client_panel_password
CLIENT_PANEL_TOKEN_SECRET=choose_client_panel_token_secret
```

If either value changes, redeploy AI Hub before testing Client Panel login.

## Environment

Create `.env.local` from `.env.example`:

```env
VITE_AI_HUB_API_BASE=http://143.198.5.97/aihub
VITE_CLIENT_PANEL_BASE_PATH=/client-panel/
VITE_DEFAULT_CLIENT_ID=ai_kart
```

Environment ownership:

- `.env.local` belongs to this project and is ignored by Git.
- AI Hub secrets stay in `AI_salesman_plugin/.env`.
- AI-KART website admin credentials stay in `Vercel_website/backend/.env`.

## Server Deployment

Use [clientpanel.md](clientpanel.md).

The deployment runbook is split into small controlled steps:

1. Safe Git pull.
2. Fast permissions.
3. Project-local Node setup.
4. PM2 setup.
5. Env file check.
6. Build.
7. PM2 restart.
8. Local smoke.
9. Public smoke.
10. Optional Hub API contract check.

Server runtime rules:

- Client Panel is Node/Vite/PM2 only.
- It does not use a Python venv.
- It does not own public Nginx.
- It does not store Hub admin secrets.
- It does not store AI-KART admin credentials.

## Git And Runtime Safety

Ignored runtime paths include:

```text
.env.local
.node/
node_modules/
dist/
.deploy-backups/
```

Deployment uses `git pull --ff-only` and stashes only tracked server-local edits. Ignored runtime files are preserved.

Do not run `git stash pop` during normal deployment unless you intentionally inspect and recover a specific stash.

## Troubleshooting

Public URL returns 404:

```text
Shared Nginx route is missing. Apply Vercel_website/aikart.md.
```

Public URL returns 502:

```text
PM2 app is down or Nginx cannot reach 127.0.0.1:5177.
```

Local smoke fails:

```bash
pm2 logs client-panel --lines 100
```

Login fails:

```text
Check CLIENT_PANEL_DEFAULT_PASSWORD and CLIENT_PANEL_TOKEN_SECRET in AI Hub .env, then redeploy AI Hub.
```

UI looks old after deploy:

```text
Run npm run build, then restart PM2 using clientpanel.md.
```
